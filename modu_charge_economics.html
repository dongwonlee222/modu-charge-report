Modupay Economics

모두페이 이코노믹스 분석

신규 사용자 획득 비용 대비 고객생애가치를 비교해 현재 마케팅 효율을 판단하고, 전체 고객과 충성 고객군의 수익 구조 차이를 보기 위한 정리본입니다.

중요 본 분석의 LTV는 순이익이 아니라 거래금액 기반으로 계산되었습니다. CPO별 마진율 차이가 반영되지 않아 실제 순이익 기준 LTV는 별도 산출이 필요합니다.

분석 프레임

분석 목적

모두페이 서비스의 마케팅 투자 대비 수익성을 검증하기 위해 신규 사용자 1명 획득 비용(CAC)과 해당 사용자가 창출하는 고객생애가치(LTV)를 비교했습니다. 핵심 질문은 현재 마케팅이 과연 회수 가능한 구조인지, 그리고 어떤 고객군이 경제성을 지탱하는지입니다.

분석 기간

  • CAC: 2024년 4월 ~ 2024년 10월
  • 전체 고객 LTV: 2022년 2월 1일 ~ 2026년 3월 11일
  • 충성 고객 기준 기간: 2026년 1월 1일 ~ 2026년 3월 11일
  • 충성 고객 기준: 월 평균 충전 1회, 결제금액 5,000원 이상

지표 정의

LTV

고객 1명이 서비스 이용 기간 동안 창출하는 총 수익 추정치입니다. 이번 분석에서는 고객당 평균 월 거래금액 × 평균 유지 개월 수로 계산했습니다.

고객당 평균 월 거래금액은 거래가 발생한 모두페이 발급 고객만을 기준으로 했고, 할인 및 포인트 사용을 제외한 실 결제 금액을 사용했습니다.

CAC

특정 기간 총 광고비를 해당 기간 신규 사용자 수로 나눈 값입니다. 광고 유입 고객이 실제 모두페이를 발급했는지 직접 추적하기 어려워 신규 사용자 수 기준으로 산정했습니다.

유지 개월 수는 모두페이 발급일에서 마지막 결제일까지의 개월 수이며, 결제가 발생하지 않은 고객은 제외했습니다.

인사이트

해석

  • 전체 사용자 기준 LTV/CAC는 높게 나타나 현재 마케팅 효율은 거래금액 기준으로는 긍정적으로 보입니다.
  • 다만 이 LTV에는 CPO별 마진율이 반영되지 않아 실제 순이익 기준 LTV는 더 낮아질 수 있습니다.
  • 충성 고객의 LTV는 비충성 고객 대비 약 8배 이상 높아, 수익이 특정 고객군에 강하게 집중되는 구조입니다.

시사점

  • 단순 신규 유입 확대보다 충성 고객 전환과 유지 전략이 더 중요한 레버입니다.
  • 마케팅 효율 검증의 다음 단계는 거래금액 기준이 아니라 순이익 기준 LTV로 내려가는 것입니다.
  • 특히 CPO 마진율, 할인, 포인트, 미납을 함께 반영한 세그먼트별 순기여 기준 대시보드가 필요합니다.

사용 SQL

전체 모두페이 이용 고객 
WITH first_card_date AS (
    SELECT user_id, MIN(create_date) AS card_create_date
    FROM everycard_issuance_history
    GROUP BY user_id
),
user_stats AS (
    SELECT
        ue.user_id,
        fcd.card_create_date,
        MAX(p.payment_log_date) AS last_payment_date,
        TIMESTAMPDIFF(MONTH, fcd.card_create_date, MAX(p.payment_log_date)) AS retention_months,
        SUM(p.cost) AS total_cost,
        COUNT(*) AS total_frequency
    FROM user_everycard ue
    JOIN first_card_date fcd ON ue.user_id = fcd.user_id
    JOIN everycard_payment_log p ON ue.id = p.everycard_id
    GROUP BY ue.user_id, fcd.card_create_date
)
SELECT
    COUNT(user_id) AS total_users,
    ROUND(AVG(retention_months), 1) AS avg_retention_months,
    ROUND(AVG(total_cost / NULLIF(retention_months, 0)), 0) AS avg_monthly_revenue,
    ROUND(AVG(total_cost / NULLIF(retention_months, 0)) * AVG(retention_months), 0) AS avg_ltv
FROM user_stats;
충성 고객 
WITH first_card_date AS (
    SELECT user_id, MIN(create_date) AS card_create_date
    FROM everycard_issuance_history
    GROUP BY user_id
),
recent_active_loyalists AS (
    SELECT
        everycard_number,
        SUM(cost) AS total_actual_cost,
        COUNT(*) AS total_tx_count
    FROM everycard_payment_log
    WHERE payment_log_date >= '2026-01-01' AND payment_log_date <= '2026-03-11'
      AND original_cost >= 5000
    GROUP BY everycard_number
    HAVING COUNT(DISTINCT MONTH(payment_log_date)) = 3
)
SELECT
    COUNT(ral.everycard_number) AS loyal_user_count,
    ROUND(AVG(TIMESTAMPDIFF(MONTH, fcd.card_create_date, '2026-03-11')), 1) AS avg_retention_months,
    ROUND(AVG(lifetime_total.total_cost /
        GREATEST(TIMESTAMPDIFF(MONTH, fcd.card_create_date, '2026-03-11'), 1)), 0) AS avg_monthly_revenue,
    ROUND(AVG(lifetime_total.total_cost /
        GREATEST(TIMESTAMPDIFF(MONTH, fcd.card_create_date, '2026-03-11'), 1))
        * AVG(TIMESTAMPDIFF(MONTH, fcd.card_create_date, '2026-03-11')), 0) AS avg_ltv
FROM recent_active_loyalists ral
JOIN (
    SELECT everycard_number, SUM(cost) AS total_cost
    FROM everycard_payment_log
    GROUP BY everycard_number
) AS lifetime_total ON ral.everycard_number = lifetime_total.everycard_number
JOIN user_everycard ue ON ral.everycard_number = ue.card_number
JOIN first_card_date fcd ON ue.user_id = fcd.user_id;
비충성 고객 
WITH first_card_date AS (
    SELECT
        eih.user_id,
        MIN(create_date) AS card_create_date,
        MAX(payment_log_date) AS last_date
    FROM everycard_issuance_history eih
    JOIN user_everycard ue ON eih.user_id=ue.user_id
    JOIN everycard_payment_log epl ON epl.everycard_id=ue.id
    GROUP BY user_id
),
all_modupay_users AS (
    SELECT DISTINCT
        ue.user_id
    FROM everycard_payment_log epl
    JOIN user_everycard ue
        ON epl.everycard_id = ue.id
),
recent_active_loyalists AS (
    SELECT
        ue.user_id
    FROM everycard_payment_log epl
    JOIN user_everycard ue
        ON epl.everycard_id = ue.id
    WHERE epl.payment_log_date >= '2026-01-01'
      AND epl.payment_log_date <= '2026-03-11'
      AND epl.original_cost >= 5000
    GROUP BY ue.user_id
    HAVING COUNT(DISTINCT DATE_FORMAT(epl.payment_log_date, '%Y-%m')) = 3
),
non_loyal_users AS (
    SELECT
        amu.user_id
    FROM all_modupay_users amu
    LEFT JOIN recent_active_loyalists ral
        ON amu.user_id = ral.user_id
    WHERE ral.user_id IS NULL
),
lifetime_total AS (
    SELECT
        ue.user_id,
        SUM(epl.cost) AS total_cost
    FROM everycard_payment_log epl
    JOIN user_everycard ue
        ON epl.everycard_id = ue.id
    GROUP BY ue.user_id
)
SELECT
    COUNT(*) AS non_loyal_user_count,
    ROUND(AVG(TIMESTAMPDIFF(MONTH, fcd.card_create_date, fcd.last_date)), 1) AS avg_retention_months,
    ROUND(
        AVG(
            COALESCE(lt.total_cost, 0) /
            GREATEST(TIMESTAMPDIFF(MONTH, fcd.card_create_date, fcd.last_date), 1)
        ),
        0
    ) AS avg_monthly_revenue,
    ROUND(
        AVG(
            COALESCE(lt.total_cost, 0) /
            GREATEST(TIMESTAMPDIFF(MONTH, fcd.card_create_date, fcd.last_date), 1)
        ) * AVG(TIMESTAMPDIFF(MONTH, fcd.card_create_date, fcd.last_date)),
        0
    ) AS avg_ltv
FROM non_loyal_users nlu
JOIN first_card_date fcd
    ON nlu.user_id = fcd.user_id
LEFT JOIN lifetime_total lt
    ON nlu.user_id = lt.user_id;
